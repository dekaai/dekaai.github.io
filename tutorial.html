Persiapan Awal

1

Membuat Kode HTML untuk Website

Sebelum memulai, siapkan terlebih dahulu kode HTML untuk website Anda. Anda bisa membuatnya menggunakan DeepSeek Chat atau editor teks lainnya.

Contoh kode HTML sederhana:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Website Saya</title>
    <style>
        body {
            font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
            line-height: 1.6;
            margin: 0;
            padding: 0;
            color: #333;
            background-color: #f9f9f9;
        }
        .container {
            max-width: 1200px;
            margin: 0 auto;
            padding: 20px;
        }
        header {
            background: #4a6fa5;
            color: white;
            padding: 2rem 0;
            text-align: center;
        }
        main {
            padding: 2rem 0;
        }
        footer {
            background: #333;
            color: white;
            text-align: center;
            padding: 1.5rem 0;
            margin-top: 2rem;
        }
    </style>
</head>
<body>
    <header>
        <div class="container">
            <h1>Selamat Datang di Website Saya</h1>
        </div>
    </header>
    
    <main class="container">
        <p>Ini adalah website pertama saya menggunakan domain sendiri.</p>
    </main>
    
    <footer>
        <div class="container">
            <p>© 2023 Website Saya</p>
        </div>
    </footer>
</body>
</html>

Simpan kode ini dengan nama index.html di komputer Anda.

Membeli Domain

2

Beli Domain di IDWebhost

Kunjungi halaman promo domain IDWebhost dan pilih domain yang Anda inginkan.

Ikuti proses pembelian hingga selesai:

  1. Cari domain yang tersedia
  2. Pilih ekstensi domain (.com, .id, .xyz, dll)
  3. Lengkapi data registrasi
  4. Selesaikan pembayaran
Catatan Penting

Tunggu hingga domain aktif (biasanya membutuhkan waktu beberapa jam hingga 1x24 jam). Periksa email Anda untuk konfirmasi aktivasi domain.

Mempersiapkan GitHub

3

Buat Akun GitHub

Buka GitHub.com dan buat akun baru.

Penting: Saat membuat user ID, gunakan huruf kecil semua (tidak boleh ada huruf besar). Contoh: usernamegithub bukan UserNameGitHub.

4

Buat Repository Baru

Setelah login, buat repository baru dengan:

  1. Klik tombol "+" di pojok kanan atas dan pilih "New repository"
  2. Beri nama repository dengan format: username-github-io (gunakan huruf kecil semua)
  3. Contoh: Jika user ID Anda dery, maka nama repository: dery-github-io
  4. Pilih "Public"
  5. Anda bisa skip bagian "Initialize this repository with a README"
  6. Klik "Create repository"
5

Upload File HTML

Di repository yang baru dibuat:

  1. Klik "Add file" > "Create new file"
  2. Beri nama file: index.html
  3. Copy-paste kode HTML yang sudah Anda siapkan sebelumnya
  4. Scroll ke bawah dan klik "Commit new file"
Tips

File index.html akan menjadi halaman utama website Anda. Pastikan kode HTML Anda valid dan lengkap.

Konfigurasi Cloudflare

6

Login ke Cloudflare

Buka Cloudflare.com dan login menggunakan akun Google Anda.

7

Tambahkan Domain ke Cloudflare

Setelah login:

  1. Klik "Add a Site" atau "Enter an existing domain"
  2. Masukkan domain yang sudah Anda beli di IDWebhost
  3. Klik "Continue"
8

Pilih Plan

Pilih plan Free dan klik "Continue"

Plan gratis Cloudflare sudah mencukupi untuk kebutuhan website pribadi atau kecil.

9

Review DNS Records

Cloudflare akan menampilkan DNS records yang terdeteksi. Anda bisa biarkan default untuk sekarang dan klik "Continue"

10

Ganti Nameserver di IDWebhost

Cloudflare akan memberikan 2 nameserver (biasanya berbentuk seperti lara.ns.cloudflare.com dan moe.ns.cloudflare.com).

Langkah-langkah:

  1. Login ke akun IDWebhost Anda
  2. Pilih domain yang Anda beli
  3. Cari menu "Nameserver" atau "Pengaturan Nameserver"
  4. Hapus nameserver default IDWebhost
  5. Ganti dengan 2 nameserver dari Cloudflare
  6. Simpan perubahan
Perhatian

Perubahan nameserver bisa memakan waktu 24-48 jam untuk propagasi, meski biasanya lebih cepat (30 menit - 2 jam).

11

Selesaikan Setup di Cloudflare

Kembali ke Cloudflare:

  1. Klik "Continue" setelah mengganti nameserver
  2. Scroll ke bawah dan klik "Check nameserver now"
  3. Tunggu hingga status domain berubah menjadi "Active"

Deployment Website

12

Setup Cloudflare Pages

Di dashboard Cloudflare:

  1. Klik "Workers & Pages" di menu sebelah kiri
  2. Pilih "Create application" > "Pages"
  3. Klik "Connect to Git"
  4. Pilih GitHub sebagai provider
  5. Izinkan Cloudflare mengakses repository GitHub Anda
  6. Pilih repository yang sudah Anda buat sebelumnya
  7. Klik "Begin setup"
13

Konfigurasi Build Settings

Pada halaman setup:

  1. Di bagian "Project name", beri nama yang sesuai
  2. Di bagian "Build settings", biarkan preset sebagai "None"
  3. Pastikan branch yang digunakan adalah "main"
  4. Biarkan build command kosong
  5. Di "Build output directory", biarkan kosong
  6. Klik "Save and Deploy"
14

Tambahkan Custom Domain

Setelah deployment selesai:

  1. Pilih tab "Custom domains"
  2. Klik "Set up a custom domain"
  3. Masukkan domain Anda (misalnya: example.com)
  4. Klik "Continue"
  5. Klik "Activate domain"
  6. Tunggu hingga status domain berubah menjadi "Active"
Selesai!

Website Anda sekarang sudah online dan bisa diakses melalui domain yang Anda beli. Proses aktivasi domain bisa memakan waktu beberapa menit hingga beberapa jam.

Troubleshooting

Masalah Umum dan Solusinya

1. Domain tidak bisa diakses setelah setup

Solusi:

  • Tunggu minimal 24 jam untuk propagasi DNS
  • Pastikan nameserver sudah benar di IDWebhost
  • Cek status domain di Cloudflare
  • Gunakan tool seperti WhatsMyDNS untuk memeriksa propagasi DNS

2. Error saat menghubungkan GitHub ke Cloudflare

Solusi:

  • Pastikan nama repository sesuai format (huruf kecil semua)
  • Pastikan file index.html ada di root repository
  • Coba disconnect dan reconnect GitHub di Cloudflare
  • Periksa izin akses GitHub di akun Cloudflare

3. Website tidak update setelah perubahan di GitHub

Solusi:

  • Tunggu beberapa menit (biasanya update otomatis dalam 2-5 menit)
  • Cek tab "Deployments" di Cloudflare Pages untuk melihat status
  • Pastikan Anda mengedit file yang benar di repository
  • Coba trigger manual deployment di Cloudflare
Bantuan Tambahan

Jika masalah masih berlanjut, Anda bisa mencari solusi di Cloudflare Community atau dokumentasi resmi GitHub dan Cloudflare Pages.
